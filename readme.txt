Основная логика приложения поделена между тремя классами:
BitMEX - взаимодействие с биржей, расчет цены
Db - чтение и запись в БД (SQLite)
Tg - telegram api

Структура БД:
таблица history - для записи событий падения цены
(timestamp INT, pair VARCHAR, price REAL, percent_change REAL)
таблица chats - список активных чатов в telegram
(chat_id INT NOT NULL)
Если файл БД отсутствует при старте, скрипт его создаст

Взаимодествие с api биржи происходит через wss протокол с использованием их же фреймворка bitmex_websocket поскольку
таким образом можно получать состояние цен в реальном времени.

После отправки сообщения о падении цены, история за последние 35с очищается дабы избежать постоянного спама, например,
в случае если цена потепенно постоянно снижается. В задании это не уточнялось и это могло быть реализовано и другим
образом в зависимости от конкретного практического предназначения скрипта. В текущем виде, например такая реализация
подходит для отслеживания, например, внезапного дэмпинга и не будет реагировать на простое снижение цены.

В процессе тестов, во время длительной работы, утечек памяти не наблюдалось и память занимаемая процессом не ревышает
20 МБ.
